Pengertian Git

GIT adalah sebuah tools bagi para programmer dan developer yang berfungsi sebagai control system untuk menjalankan proyek pengembangan software. Tujuan penggunaan GIT yakni untuk mengelola versi source code program dengan menentukan baris serta kode yang akan ditambahkan atau diganti. GIT terkategori dalam DVCS (Distributed Version Control System), yang artinya kode tidak hanya memiliki satu tempat penyimpanan sejarah lengkap sebuah software. Setiap salinan kode kerja developer juga berperan sebagai repositori yang dapat berisi riwayat lengkap dari semua perubahan.

Version Control System

Version Control System adalah sebuah kategori software mengelola perubahan pada source code. Version control system mencatat semua modifikasi yang dilakukan ke kode di sebuah database khusus. Jika terjadi sebuah kesalahan, developer bisa melihat kode versi awal untuk memperbaiki kesalahan dengan disrupsi minimal. Version control melindungi source code dari berbagai masalah atau kesalahan manusia dan konsekuensi yang tidak diinginkan. Developer software terus menulis source code baru dan mengubah source code yang ada.

Perbedaan Git dan Github

1. Bentuk Pengoprasian

Git merupakan perangkat lunak open source yang dirancang untuk merangkum riwayat sumber dari coding. Sedangkan Github merupakan layanan hosting berbasis web sebagai repository Git.

2. Menyertakan Perintah-Perintah yang Berbeda.

Git menangani perintah yang berbeda dan cenderung fokus pada tugas-tugas SCM yang eksklusif seperti push and pull, reset, commit, fetch, dan marge. Sedangkan Github berfungsu sebagai tuan rumah terhadap repository Git dalam penyimpanan kode di lokasi yang ringkas.

Perintah Dasar Git

1. GIT init: Membuat sebuah repository yang berbeda pada file lokal dimana nantinya file tersebut bernama .git.

2. GIT status: Mengetahui status dari sebuah repository lokal.

3. GIT add: Menambah file baru yang ada pada repository yang telah dipilih.

4. GIT commit: Menyimpan perubahan yang telah dilakukan.

5. GIT push: Digunakan untuk mengirimkan perubahan file setelah di commit di remote repository.

6. GIT branch: Melihat seluruh branch yang ada dalam repository.

7. GIT checkout: Menukar branch yang aktif dengan branch yang telah dipilih.

8. GIT marge: Menggambungkan branch yang aktif dan yang telah dipilih.

9. GIT clone: Membuat salinan repository lokal.
